## 🚀 安装到浏览器

### Chrome/Edge 浏览器

1. **打开扩展管理页面**
   - Chrome: 地址栏输入 `chrome://extensions/`
   - Edge: 地址栏输入 `edge://extensions/`

2. **启用开发者模式**
   - 右上角打开"开发者模式"开关

3. **加载扩展**
   - 点击"加载已解压的扩展程序"
   - 选择插件目录：`/Users/tonysheng/Desktop/编程/插件/book-splitter-extension`
   - 点击"选择文件夹"

4. **验证安装**
   - 扩展列表中出现"书籍章节拆分工具"
   - 工具栏显示扩展图标（蓝色书籍图标）

## 📚 使用方法

### 方式一：直接使用扩展

1. 点击浏览器工具栏中的扩展图标
2. 点击"选择书籍文件"按钮
3. 选择 PDF、EPUB 或 MOBI 文件
4. 配置拆分选项：
   - ✅ 自动检测章节（推荐）
   - 选择章节识别模式
   - 选择输出格式（txt/md/html）
5. 点击"开始拆分"按钮
6. 等待处理完成
7. 点击"下载所有章节"保存文件

### 方式二：在 NotebookLM 中使用

1. 访问 [NotebookLM](https://notebooklm.google.com)
2. 页面右下角自动显示"📚 拆分书籍"浮动按钮
3. 点击按钮后会提示你点击扩展图标
4. 按方式一的步骤操作即可

## ⚙️ 功能特性

### 支持的文件格式
- **PDF** - 完整支持，自动提取文本
- **EPUB** - 完整支持，按章节结构拆分
- **MOBI** - 基础支持（建议转换为 EPUB）

### 章节识别模式
1. **自动检测** - 智能识别多种章节格式
2. **数字编号** - 匹配"第X章"、"Chapter X"
3. **中文标题** - 匹配"第一章"、"第二章"
4. **罗马数字** - 匹配"I"、"II"、"III"
5. **自定义正则** - 使用正则表达式精确匹配

### 输出格式
- **.txt** - 纯文本格式
- **.md** - Markdown 格式（推荐用于 NotebookLM）
- **.html** - HTML 网页格式

## ⚠️ 注意事项

### 首次使用
1. 需要网络连接（首次加载 PDF.js 和 JSZip 库）
2. 大文件处理可能需要几分钟时间
3. 建议从小文件开始测试

### 文件建议
- **PDF**: 最好是基于文本的 PDF（非扫描版）
- **EPUB**: 标准格式的 EPUB 文件
- **MOBI**: 建议先用 Calibre 转换为 EPUB

### 章节识别
- 如果自动检测失败，尝试其他识别模式
- 可以使用自定义正则表达式精确匹配
- 不同的书籍格式可能需要不同的模式

## 🐛 故障排除

### 问题：扩展无法加载
**解决方法**：
- 确保启用了开发者模式
- 检查目录路径是否正确
- 查看扩展页面的错误信息

### 问题：处理失败
**解决方法**：
- 检查网络连接（需要访问 CDN）
- 确认文件格式正确
- 尝试较小的文件
- 查看浏览器控制台的错误信息

### 问题：章节识别不准确
**解决方法**：
- 尝试不同的识别模式
- 使用自定义正则表达式
- 检查书籍的章节标题格式

### 问题：下载被阻止
**解决方法**：
- 允许浏览器下载多个文件
- 检查下载权限设置
- 尝试逐个下载章节

## 📖 示例：自定义正则表达式

### 常见章节格式

```
# 中文数字章节
^第[一二三四五六七八九十]+章[：:]

# 数字章节
^第\d+章

# 英文章节
^Chapter\s+\d+

# 带副标题的章节
^第[一二三四五六七八九十\d]+章\s+(.+)
```

## 🎯 最佳实践

1. **文件选择**
   - 优先使用 EPUB 格式（结构最清晰）
   - PDF 应选择有明确章节标题的
   - 避免使用扫描版 PDF

2. **章节识别**
   - 先使用自动检测模式
   - 如果效果不好，查看书籍的实际章节格式
   - 使用自定义正则精确匹配

3. **输出格式**
   - NotebookLM 推荐 Markdown 格式
   - 需要打印选择 HTML 格式
   - 纯文本最兼容但格式较少

4. **批量处理**
   - 建议一次处理一个文件
   - 大文件可以分批处理
   - 处理完成后立即下载
